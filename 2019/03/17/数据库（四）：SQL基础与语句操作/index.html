SQL语言概述

SQL语言的特点

    SQL 是用于访问和处理数据库的标准的计算机语言。如今比较常用的数据库大多为关系型数据库，这类数据库包括：MySQL、SQL Server、Access、Oracle、Sybase、DB2 等等。

  • 一体化：
  • 高度非过程化
  • 简洁
  • 使用方式多样
    SQL语言主要功能
  • 数据查询功能
  • 数据定义功能
  • 数据操纵功能
  • 数据控制功能
SQL功能 命令动词
数据查询 SELECT
数据定义 CREATE DROP ALTER
数据操纵 INSERT UPDATE DELETE
数据控制 GRANT REVOKE DENY

    其中，数据定义功能用于定义（CREATE）、删除（DROP）、修改（ALTER）数据库中的对象（关系表、视图、索引）；数据查询用于查询数据的功能，虽然只有SELECT一条命令，但是在熟练的掌握之后，SELECT的功能是十分强大的，他可以进行数据的筛选，只要提供了正确的搜索条件，你可以熟练使用你的数据库；数据操纵可以增加、更新、删除数据库中的数据，数据操纵功能与定义功能不同的地方在于前者的操作是对于数据库的数据而言的，而后者则是针对数据库对象来说的；数据控制功能可以控制用户对数据的操作权限。

SQL语言支持的数据类型

    SQL中的数据主要包括以下几种类型：

  • 数值型
  • 字符串型
  • 日期时间型
  • 货币型

    不同的数据库可供选择的数据类型是不全一致的，因此在使用数据库进行数据的使用时，应尽量使用标准SQL库中支持的数据类型，方便后期的项目移植以及多平台使用。下面是菜鸟教程中介绍的常用的SQL数据类型：SQL数据类型。
其中字符串型有三类组成，分别是：

  • 普通编码字符串类型
  • 统一字符编码字符串类型
  • 二进制字符串类型

SQL基础语法使用

数据定义功能

    在关系数据模型中，一般将数据划分为三个层次，外模式–>视图，模式–>表，内模式–>索引，在这三个模式中，模式与另外两个建立起两层联系，方便了数据库的管理与存储。另外一个数据库对象——架构，则相当于容器，可以在其下面创建表、视图等数据库对象。如果把数据库系统比喻成操作系统，那么架构则对应于目录，存放的其他对象就好比一个个文件。
| 数据库对象 | 创建 | 修改 | 删除 |
| ———- | ————- | ———– | ———– |
| 架构 | CREATE SCHEMA | 架构不可修改 | DROP SCHEMA |
| 表 | CREATE TABLE | ALTER TABLE | DROP TABLE |
| 视图 | CREATE VIEW | ALTER VIEW | DROP VIEW |
| 索引 | CREATE INDEX | ALTER INDEX | DROP INDEX |

基本表操作

    表的完整性约束：一般分为列级完整性约束和表级完整性约束，使用完整性约束可以增加数据的约束与数据之间的关联。

NOT NULL 指定列不为空 只可用于列级约束
PRIMARY KEY 主键约束（可以理解为表的一个标记项）
FOREIGN KEY 外键约束（引用其他表的外键）
UNIQUE 唯一值约束（可以存在一个NULL值）
DEFAULT 默认值约束（指定列的默认值） 只可用于列级约束
CHECK 列取值范围约束（指定列的取值范围或者多列之间的关系） MySQL对这个约束并不是很支持
